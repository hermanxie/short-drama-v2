# 动作提示词策略探索 v0

## 起点判断

上一级主项目的稳定经验仍然有效：

- 草图提示词和视频提示词必须来自同一份结构化蓝图。
- 动作 block 第一帧必须直接进入动作。
- 草图可以包含宫格、编号、箭头、镜头参数和图例。
- 视频提示词必须排除草图线稿、编号、箭头、UI、水印、字幕和说明文字。
- 人物、场景、道具应以 manifest 或明确输入为准。
- 视频提示词必须重视脸、眼、嘴、手、关键道具和光影对焦。
- 新策略必须继承原有系统提示词的风格与画质优势，尤其是人物脸部和手部补光、皮肤质感、立体感、眼神光、轮廓光和关键道具高光。
- 画面文字禁令需要比普通提示更强：字幕、台词字幕、旁白文字、镜头参数文字、角色名标签、UI、水印、平台 logo、时间码、分镜编号、底部图例、箭头说明、构图辅助线、草图宫格边框都不得进入最终视频画面。

## 新方向

本实验区尝试把“动作”作为提示词生成的第一层结构，而不是只从剧情摘要进入。

初步拆解维度：

- 动作目的：攻击、防守、追逐、闪避、递接、冲撞、施法、显现、转身、坠落。
- 身体动力：重心、发力点、方向、速度、停顿、反作用力。
- 空间关系：人物距离、朝向、高低位、前后景、遮挡关系。
- 接触点：手、脚、武器、道具、衣摆、墙面、地面、门、能量边界。
- 环境反馈：尘土、碎石、衣料、火光、风沙、光影、地面痕迹。
- 镜头任务：让观众看清动作因果，而不是只制造气氛。

## 打斗动作新增硬约束

动作不能发生在空白背景里。只要进入打斗，每个动作镜头都要把环境变成动作的一部分：

- 环境可以提供借力点，例如柱子、墙面、台阶、栏杆、铁链、门框。
- 环境可以制造阻碍或遮挡，例如雕塑、桌椅、烟尘、碎石、布帘、尸体。
- 环境可以承受冲击，例如地板被砸裂、墙面被刀锋擦出火星、雕塑被撞碎、栏杆被压弯。
- 环境可以改变动作路线，例如绕柱旋转、借高差下压、贴墙滑步、穿过门框封路。

动作必须是连续链条，不写单招摆拍。每个动作镜头至少要形成：

```text
起手 → 接触 → 受力反馈 → 换位/反击 → 封口结果
```

时间轴提示词中应尽量保持 0-3 秒、3-6 秒、6-9 秒、9-12 秒、12-15 秒连续推进，让人物始终在移动、反应、变招和压迫中。
