# 动作优先提示词规则 v1

## 核心原则

本实验区采用 Action-First Prompting：先设计动作因果，再生成镜头提示词。

动作提示词不从氛围开始，而从人物目的、身体动力、空间关系、环境参与和命中结果开始。

本实验区的单镜提示词结构以 `03_templates/星辰剧本到提示词.md` 的完整镜头格式为基础，不得简化为摘要式提示词。必须保留“故事定位、核心视觉统一、角色统一设定、分镜规划清单、挂载、相机位置、相机朝向、角色朝向、构图锚点、起幅、落幅、连续性约束、提示词、镜头参数、抽针/甩拍/短爆、微表情、钉子4行”的骨架，并在此基础上增强全剧理解、镜头衔接、环境动态、风格美学和高风险规避。

## 全剧理解与镜头衔接

生成单集或单镜提示词前，必须先理解剧本全局，至少建立以下信息：

- 本集在全剧中的位置。
- 本集核心冲突和情绪推进。
- 角色关系、阵营变化和心理状态。
- 人物伤势、脏污、能量、道具状态的连续性。
- 场景空间轴线、入口出口、远近层次和群演调度。
- 上一镜如何落下，下一镜如何接起。

每个镜头必须显式写清：

- 上一镜承接点。
- 本镜起幅。
- 本镜落幅。
- 下一镜过门钩子。
- 人物位置变化。
- 道具、伤势、战损或能量状态变化。

禁止把 15 个镜头写成互不相干的漂亮片段。镜头之间必须形成连续的戏剧推进和空间推进。

## 必须输入

生成前必须先收集：

- 本集脚本。
- 角色参考图和角色设定。
- 场景参考图和空间结构。
- 道具、武器、服装参考。
- 平台限制，例如时长、字数、比例、是否需要分镜图。

如果图片缺失，应先说明缺失影响，不应假装已经参考。

## 动作镜头硬规则

凡涉及打斗动作，必须满足以下条件：

- 第一秒直接进入动作，不用平静站姿铺垫。
- 环境必须参与动作，不得只是背景。
- 每个动作镜头必须形成连续动作链。
- 命中点必须有接触、短停、受力反馈和结果。
- 镜头运动必须服务于动作可读性。
- 禁止只写气氛、光效、爆炸、能量，不写动作因果。
- 动作表演必须炫酷但可读，必须看清“谁对谁做了什么、在哪里接触、如何受力、结果是什么”。
- 强动作必须有身体发力逻辑，不能只靠特效或镜头晃动制造刺激。

## 环境参与规则

每个打斗镜头至少选择一种环境功能：

- 遮挡：柱子、墙、门框、烟尘、布帘、雕塑遮住视线并制造换位。
- 借力：人物踩墙、踏台阶、抓铁链、撑栏杆、绕柱发力。
- 撞击：身体撞墙、撞柱、砸桌、压碎雕塑、撞断栏杆。
- 破坏：武器砸裂地板、刀锋擦墙出火星、碎石飞溅、木屑炸开。
- 高差：台阶、平台、深井、桥面、坡面制造上压下攻或低闪穿线。
- 封路：门框、倒塌物、尸体、碎石、栏杆限制追击路线。
- 节奏切换：靠柱喘息、滑步刹停、碎物落地、铁链余震形成短促停顿。

## 连续动作链

每个动作镜头必须按链条设计：

```text
起手 -> 接触 -> 受力反馈 -> 换位/反击 -> 封口结果
```

15 秒镜头建议拆成：

```text
0-3秒：直接起手，建立攻击方向和环境参与点。
3-6秒：第一次有效接触，出现短停、受力反馈和空间位移。
6-9秒：对手回应或主角变招，利用环境完成换位。
9-12秒：主爆点，动作升级但必须可读。
12-15秒：封口结果，保留下一镜钩子。
```

## 镜头语言规则

动作镜头优先使用：

- 35mm 广角动作视角。
- 低机位贴地前冲。
- 贴身高速跟拍。
- 短横移或短弧半环绕。
- 命中帧后回正侧位或斜正位确认结果。
- 极端低角度制造压迫。
- 前景遮挡制造空间穿插。
- 长焦压缩强化逼近感。
- 斜线构图、动态对角线和高低差构图。
- 鞭甩转场只用于换位或遮挡切位，甩完必须快速回稳。

限制使用：

- 慢镜。
- 纯特写。
- 长时间乱抖。
- 无目标大环绕。
- 特效遮挡命中点。
- 武器奇异拉长。

## 段落数量不限

最终视频提示词的时间轴段落数量必须根据剧情、动作密度和情绪转折自由切分，不得机械套用固定三段式或固定五段式。

- 复杂动作可切为 `0-2秒 / 2-4秒 / 4-6秒 / 6-9秒 / 9-12秒 / 12-15秒` 等更多段。
- 简单单动作可以只切 1-2 段。
- 单镜到底可以只写一个连续时间轴，但必须写清起承转合。
- 每段必须承担不同调度目的，禁止重复同一姿势或只改变镜头距离。

## 环境动态强制规则

每条最终视频提示词必须包含环境动态描述。环境动态不只是背景氛围，而要参与动作、塑造速度、表达受力或提供空间证据。

可使用但不限于：

- 体积雾流动。
- 沙尘飞扬。
- 脚底沙尘受重力影响扬起。
- 空中光影粒子流动。
- 火光摇曳闪烁。
- 风压推动衣摆和发丝。
- 碎石坠落。
- 铁链震动。
- 地面裂纹扩散。
- 建筑残骸受冲击崩落。
- 武器擦墙火星、砸地碎裂、撞柱回响。

如果一个镜头没有环境动态，必须重写。

## 风格与画质继承规则

新提示词必须继承原有系统提示词中关于风格、画质、光影和人物质感的优点。

视频画面应保持：

- 真人写实电影画面。
- 现场拍摄感。
- 清晰高画质。
- 人物皮肤有真实纹理和柔和立体感。
- 脸部、眼睛、嘴唇、手部、关键道具优先对焦。
- 强方向主光塑造身体轮廓和空间层次。
- 人物脸部和手部必须有清晰柔和补光，避免欠曝、灰脸、五官糊。
- 眼神光明确，鼻梁、颧骨、嘴唇、手背、武器边缘有可读高光。
- 轮廓光把人物从背景中剥离。
- 背景只承担纵深、压迫、战场氛围和环境互动，不抢人物焦点。
- 8K 超高清、电影感、物理真实感、画面稳定无闪烁。
- 粗粝胶片颗粒质感，低饱和胶片色，真实现场拍摄质感。
- 强方向光、侧光、伦勃朗光或明暗对照必须服务于人物面部骨骼立体感。
- 即使逆光，也必须有面部和手部精准柔和补光。
- 皮肤保留微观毛孔、细微绒毛、微汗和真实瑕疵，不得过度磨皮。

禁止出现：

- 平光。
- 灰雾糊脸。
- 脸部欠曝。
- 手部糊成一团。
- 背景抢焦。
- 皮肤塑料感。
- 过度磨皮。
- 只写动作不写人物质感和光影层次。

## 画面文字禁令

视频提示词必须明确禁止任何文字进入最终画面。

画面中严禁出现：

- 字幕。
- 台词字幕。
- 旁白文字。
- 说明文字。
- 分镜编号。
- 镜头参数文字。
- 角色名标签。
- UI。
- 水印。
- 平台 logo。
- 时间码。
- 底部图例。
- 箭头说明。
- 构图辅助线。
- 草图宫格边框。

剧本文字、台词、镜头参数、钉子4行、审核说明只能作为生成指令，不得被视频模型渲染成画面内容。

每条最终视频提示词末尾必须加入完整的风格与画质约束、完整的画面文字禁令。不得只在文件顶部写通用规则，也不得在单镜提示词中写“遵守通用约束”之类的引用式短句。每个单镜提示词被单独复制到平台时，都必须自带禁止字幕、禁止文字、禁止 UI、水印和说明标记的完整约束。

## 对白与声音规则

如果剧本中有台词，必须写入对应时间轴段落，不得遗漏。

台词格式建议：

```text
台词（人物名）：（情绪表情 + 面部肌肉动作 + 口型节奏）“台词具体内容”。
```

台词只作为角色声音、口型、呼吸和表演指令，不允许画面中出现字幕、台词文字、对白气泡、旁白文字或任何可读文字。

声音设计应保持现场收音质感：环境音、物理拟音、脚步、衣料摩擦、金属碰撞、风沙、碎石、铁链、呼吸、怒吼可以出现；除非用户明确要求，禁止背景音乐和旋律。

## 高风险规避 / 反向约束

每条最终视频提示词必须包含高风险规避，不得只在全局说明中写一次。

必须明确拒绝：

- 字幕、台词字幕、旁白文字、说明文字、角色名标签、UI、水印、平台 logo、时间码。
- 塑料质感、3D 渲染感、CGI 感、虚幻引擎感。
- 物理规则崩坏、重力异常、人物漂浮、武器拉长。
- 肢体多余、手指融合、脸手失焦、动态模糊导致画质降级。
- 平光、无方向漫反射散光、面部缺乏阴影、脸部欠曝。
- 过度磨皮、美颜滤镜、皮肤蜡像感。
- 建筑结构融化、比例扭曲、场景主要特征丢失。

## 质检钉子

每个动作镜头输出后必须附四行质检：

```text
第一关键接触闭环：
空间尺度与纵深证据：
伤势/失衡/心理递进：
封口落幅与下一镜钩子：
```

如果四行写不出来，说明该镜头动作因果还不够清楚，必须重写。
